# v3.2.2 real Loop control incident regression

Date: 2026-07-14

## Scope

This record is a sanitized regression summary derived from a stopped real
Adaptive Loop. Raw rollout JSONL, databases, generated Pack contents, provider
keys, prompts, and model responses are intentionally excluded.

The observed failures were:

- a PTY/state-write path started a fixed-byte stdin helper before the runtime,
  so the runtime's bounded reader never started;
- a deferred external model call completed but its stdout was unavailable;
- two freshness/control-plane closures were counted as product repair attempts;
- the Controller received a new Pack while canonical state still named the old
  digest;
- one long App turn acquired multiple successive route leases.

## Regression contract

`tests/test_real_incident_regression.py` proves that:

- a freshness `BLOCKED` result with `execution_started=false` is durable but
  does not consume repair;
- initial execution plus three repairs and two zero-execution closures still
  permits the remaining authorized repairs;
- a completed external-call receipt survives discarded stdout and a lone
  started receipt has conservative call accounting;
- a changed Pack digest is rejected until an atomic canonical migration;
- a routed legacy state without Controller-turn fields is deterministically
  backfilled before migration enables turn enforcement;
- a single `controller_turn_id` cannot acquire a second route lease;
- unsafe pre-runtime stdin helpers are rejected by the Pack validator.

## Claim boundary

This evidence covers the loop-skill runtime, schemas, renderer, validator, and
local subprocess harness. It does not claim that Codex app-server now terminates
orphaned process groups, provides an independent automation emergency channel,
or behaves identically across all App versions.
