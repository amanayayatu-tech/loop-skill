# v3.2.1 payload transport and repair-exhaustion evidence

Date: 2026-07-14
Scope: repository runtime/Pack protocol plus bounded Codex App smoke
Claim boundary: not an upstream Codex app-server process-reaping fix

## Sanitized incident signal

A real Adaptive Loop attempted to materialize an approximately 8 KB payload
through a fixed-byte stdin pipeline. The producer did not deliver or close the
full frame, leaving both the fixed-byte reader and Python runtime waiting. The
children remained attached to app-server and coincided with task-read and
automation-pause timeouts. Restarting the App reaped the processes. Raw rollout
content, databases, credentials, user project state, and the generated Pack are
intentionally excluded from this repository.

This evidence supports a transport-deadlock hypothesis, not proof that the
pipeline was the only cause of every app-server timeout.

## Codex App canary

The canary must record only sanitized identities and results:

1. One 8–16 KB non-PTY payload materializes successfully.
2. One deliberately partial frame reaches the runtime's 30-second structured
   timeout; its session exits and no matching runtime child remains in `ps`.
3. Task read and automation pause remain responsive after the timeout.
4. A non-Git fixture with repair limit 1 follows the Decision Card fast-stop
   path and reaches canonical `FINALIZATION_ACKED`.

Results are appended here only after execution. This canary is App smoke
evidence; it does not claim that app-server now creates and reaps independent
process groups.

## Results

### Bounded runtime and App smoke

- A direct non-PTY runtime child received one compact 12,417-byte JSON frame
  while the parent deliberately kept stdin open. It returned exactly one
  `PAYLOAD_MATERIALIZED` object with exit code 0 in 0.038 seconds.
- A second non-PTY runtime child received only `{"envelope_type":` while stdin
  remained open. It returned `INPUT_TRANSPORT_TIMEOUT` with exit code 1 after
  30.072 seconds; the parent reaped the PID and a subsequent existence probe
  returned `ProcessLookupError`.
- Codex App task suffix `…73b68` executed the repair-limit-1 non-Git fixture.
  The named test returned exit code 0, printed `OK`, and asserted the canonical
  terminal operation status `FINALIZATION_ACKED`.
- After the partial-frame timeout, App task-read completed immediately. A
  dedicated temporary heartbeat was created, changed from `ACTIVE` to `PAUSED`,
  verified from its persisted automation record, and deleted successfully.

These results are current-machine App smoke only. Repository release gates and
remote CI results are recorded below after their final execution.
